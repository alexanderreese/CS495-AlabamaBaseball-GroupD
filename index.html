Pitch Grader

Project Description

University of Alabama Baseball team wants a tool to rate baseball pitches.
They want this done using statistics gathered from the UA baseball team as well as other NCAA college baseball teams.
Using these statistics, the tool should rate each pitch a pitcher throws.

Client Needs

Needs:

 Automate quantifying a rating for pitches
 Grade 26 possible pitches
 Output a rating

Wants:

 Ability to change the grading criteria
 Ability to insert multiple pitches by uploading an excel sheet

Sprint 1

Our goals for this sprint were to develop a standardized system to rate the pitches, and to develop our clickable prototype.
We achieved both of these goals and also decided to recreate our project in React for more versatility.

Sprint 2

Our goal for this sprint was to develop the React prototype and add our calculation methods to each pitch.
We were able to develop the React prototype, but the calculations proved to be very time consuming and we had to push that goal to sprint 3.
In place of our unachieved goal, we added graphs for visual representation of the pitch ratings, got an updated list of client needs and wants.
We also began developing a python backend for simplicity of calculation.

Sprint 3

Our client updated his list of needs to include the ability to enter multiple pitches at once, produce graphs for each entered pitch, output an arsenal score, and add a settings page to change the rating scales of each pitch.
In addition to the new list of needs, the client had a list of less important “wants”. These wants included the ability to read in pitches through an excel file, the ability to generate pdf reports of each pitch/pitcher, and to add feedback to the pitcher based on their metrics.
Taking in this updated list of goals, we divided up the tasks as follows:

* -> Excel parser and pdf generator: Tyler McWilliams
* -> Calculations of all pitches: Will Miller
* -> Settings page design and implementation: Luke Lindsay
* -> Frontend/backend communication/design: Alex Reese
* -> Pitcher feedback: All
We were able to achieve every goal our client set, with the exception of the pitcher feedback. We decided that this feedback would require a more in depth understanding of pitching than any of us understood, and the client would not have the time to help us develop that function.
The greatest difficulty we faced during this sprint was implementing communication between the JavaScript/React front and the Python back end, developing a method of calculations in the python backend, developing different methods of grading based on the goal of the pitch (whether it should be as high as possible or be as close to a certain score as possible), and finally learning to parse excel files and format PDF reports.
Overall we are happy with how our project turned out, and the client was ecstatic to receive more functionality and an overall more advanced program than he had originally anticipated.
